# 產品需求文件 (PRD)

## 專案簡介
本專案旨在開發一個具備前後端架構的 AI 聊天機器人。除了基本的對話功能外，也整合了多對話工作階段 (Session)、多模態輸入 (檔案上傳)、記憶機制以及工具呼叫能力，提供使用者更進階且個人化的 AI 體驗。

## 目標使用者
- 需要管理多個獨立對話脈絡的使用者。
- 希望 AI 能夠記憶個人偏好或特定資訊的使用者。
- 需上傳圖片或文件讓 AI 進行分析的使用者。

## 核心功能清單
1. **對話狀態管理 (Session)**
   - 支援建立多個獨立的聊天室。
   - 每個聊天室維持獨立的上下文 (Context)。
2. **訊息系統**
   - 記錄每一則訊息的傳送者 (Role: user/assistant)。
   - 記錄訊息內容 (Content) 與時間戳記 (Timestamp)。
3. **對話歷史管理**
   - 顯示歷史 Session 列表。
   - 允許切換不同 Session 並載入過往對話。
4. **上傳圖片或文件**
   - 支援使用者上傳圖檔或文件作為對話輸入。
   - 前端需提供上傳介面與預覽。
5. **回答控制**
   - 重新生成 (Regenerate)：捨棄上一次 AI 回應，重新要求生成。
   - 停止生成 (Stop)：中斷正在進行的生成過程。
6. **記憶機制 (Memory)**
   - 儲存使用者的偏好或指定資訊。
   - 實現跨 Session 的記憶提取，讓 AI 可以提供個人化回應。
7. **工具整合 (Tool Integration)**
   - 串接外部 API 或模擬工具 (例如：查詢時間、網路搜尋)，讓 AI 具備實際操作與獲取即時資訊的能力。

## 非功能性需求
- **易用性**：前端介面直覺，具有良好的回應速度與 RWD (響應式網頁設計)。
- **可擴展性**：後端架構允許輕鬆新增新的 Tool 或修改 Prompt。
- **安全性**：確保 API Key 不外洩 (使用環境變數)。
